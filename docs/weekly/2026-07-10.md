### llm随机性

llm生成文字的过程，本质就是从候选词里面随机挑选。llm有一个temperature参数，temperature=0，确定性。
temperature = 0：永远选概率最高的（确定性）

temperature = 1（默认）：按概率分布随机选，60% 的时候选第一个，35% 的时候选第二个

temperature = 2：更随机，低概率的也有可能被选中

GLM-4-flash 默认 temperature 不是 0，所以同样的问题，今天可能选 RAG，明天可能选联网。

怎么解决？
设 temperature = 0（但智谱 API 不一定支持）

**多跑几次取多数投票**（rounds 3 就是干这个的）

| 问题 | 现状 | 解决方案 | 什么时候做 |
|------|------|---------|-----------|
| 隐私性 | 用户数据发给智谱云API | 本地模型 / 脱敏处理 / 私有化部署 | 长期 |
| 模型不够聪明 | GLM-4-flash免费模型，工具选错 | 加Planner+Reflection / 换更强模型 | 正在做 |
| 法律准确性 | 可能给错建议 | 强制引用法条 / 免责声明 / 限制范围 | Planner阶段 |
| 知识库覆盖面 | 只有消费者权益保护法1部 | 扩充更多法律PDF | 长期 |
| 响应速度 | 平均11秒，用户等不了 | 流式返回 / GPU加速 / 缓存 | UI阶段 |
| 会话持久化 | 存内存，重启就丢 | 换SQLite/Redis | 长期 |
| Agent不够像律师 | 不会追问，给通用回答 | Planner信息收集 / 更多工具 / 长期演进 | 正在做+长期 |

现在的逻辑：

全部 round 成功	"ok"	正常参与统计

部分 round 报错	"partial"	用成功的 round 投票，参与统计

全部 round 报错	"error"	排除在准确率之外，单独报告
输出示例：

工具选择准确率: 8/8 = 100.0%
  ⚠️  2 题因超时/报错被排除，不计入准确率

这样准确率反映的是 *Agent 真实决策能力*，*超时/服务器问题不会污染指标*。
